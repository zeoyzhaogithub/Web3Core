# swap数学

## 概念

Uniswap 的 Swap 数学核心在于其自动化做市商 (AMM) 模型，特别是 恒定乘积公式 x * y = k。以下是对 Swap 过程的详细数学解析：

## 1. 核心公式：恒定乘积 x * y = k

x: 交易池中 Token A 的数量 (储备金)。

y: 交易池中 Token B 的数量 (储备金)。

k: 恒定乘积常数。在不考虑手续费的理想情况下，Swap 操作前后 k 值保持不变。

核心思想：任何交易都必须确保交易后 (x_new) * (y_new) = k。交易导致 x 和 y 的变化方向相反（买入 A 会减少 A 的储备，增加 B 的储备，反之亦然）。

## 2. Swap 过程详解 (以用 Token A 买入 Token B 为例)

用户想将 Δx 数量的 Token A 兑换成 Token B。

### 步骤 1: 扣除手续费 (实际进入池子的数量)

Uniswap 收取一定的交易手续费 (例如 V2/V3 通常为 0.3%)。

实际添加到池子中的 Token A 数量是 Δx * (1 - fee)。

设手续费率为 φ (例如 0.003 表示 0.3%)，则：
Δx' = Δx * (1 - φ)

### 步骤 2: 计算交易后新的储备金

- 交易前储备金： x (Token A), y (Token B), k = x * y

- 用户添加 Δx' 个 Token A 到池子，新的 Token A 储备金变为： x_new = x + Δx'

- 根据恒定乘积公式，新的 Token B 储备金 y_new 必须满足：
x_new *y_new = k
(x + Δx')* y_new = x * y

- 解出 y_new：
y_new = (x * y) / (x + Δx')

### 步骤 3: 计算用户收到的 Token B 数量 Δy

- 用户收到的 Token B 数量等于交易前后 Token B 储备金的减少量：
Δy = y - y_new

- 将 y_new 代入：
Δy = y - [(x * y) / (x + Δx')]

- 化简：
Δy = y *[1 - (x / (x + Δx'))] = y* [(x + Δx' - x) / (x + Δx')] = y * [Δx' / (x + Δx')]

### 步骤 4: 最终输出公式 (用户收到的 Token B 数量)

Δy = (y * Δx') / (x + Δx')

关键点：

- Δy 是用户最终得到的 Token B 数量。

- Δx' 是扣除手续费后实际进入池子的 Token A 数量 (Δx' = Δx * (1 - φ))。

## 3. 价格与滑点 (Slippage)

### 瞬时价格 (Spot Price)

当前池子中 Token A 相对于 Token B 的瞬时价格为 P = y / x (即 1 个 A 能兑换多少 B)。

这个价格仅对无限小的交易量有效。任何实际交易量 Δx > 0 都会导致执行价格变化。

### 有效价格 (Effective Price)

- 用户实际执行这次交易的有效价格为：
Effective Price = Δy / Δx

- 由于 Δy < (y / x) * Δx (因为分母 (x + Δx') > x)，所以 Effective Price < P。用户得到的价格比交易前看到的瞬时价格差。

### 滑点 (Slippage)

- 滑点衡量有效价格与瞬时价格的偏离程度：
Slippage = (P - Effective Price) / P = 1 - (Effective Price / P)

- 将公式代入：
Effective Price / P = (Δy / Δx) / (y / x) = (Δy *x) / (Δx* y)

- 再代入 Δy = (y *Δx') / (x + Δx') 和 Δx' = Δx* (1 - φ)：
Effective Price / P = [((y *Δx') / (x + Δx'))* x] / (Δx *y) = [x* Δx' *x] / [(x + Δx')* Δx *y]* (y / x) (化简过程略)

- 最终可得：
Effective Price / P = (x *Δx') / [(x + Δx')* Δx] = (Δx' / Δx) *[x / (x + Δx')] = (1 - φ)* [1 / (1 + (Δx' / x))]

- 所以滑点取决于两个关键因素：

  - 交易量相对于储备金的大小 (Δx' / x): 交易量越大 (占池子比例越高)，滑点越大。这是分母 (x + Δx') 显著大于 x 的直接结果。

  - 手续费率 (φ): 手续费进一步降低了用户获得的价格。

## 4. 重要公式总结 (买入 B 为例)

### 实际进入池子的输入量: Δx' = Δx * (1 - φ)

### 用户收到的输出量: Δy = (y * Δx') / (x + Δx')

### 交易后新的储备金

x_new = x + Δx'

y_new = y - Δy = (x *y) / (x + Δx') (根据 x_new* y_new = k)

### 新的恒定乘积常数 (k'): k' = x_new *y_new = (x + Δx')* [(x *y) / (x + Δx')] = x* y = k (手续费使 k' 略微增大！)

- 手续费的影响: 虽然交易基于 x *y = k 计算，但因为用户投入的 Δx 只有 Δx' 进入池子，而 Δy 是足额支付的，导致新的乘积 k' = x_new* y_new = (x + Δx') *(y - Δy)。将 Δy 公式代入会发现 k' = (x + Δx')* [y - (y *Δx')/(x + Δx')] = (x + Δx')* y *[1 - Δx'/(x + Δx')] = (x + Δx')* y *[x/(x + Δx')] = x* y = k。手续费收入体现在池子总价值 (x_new * P + y_new 或按外部价格计算) 的增加上，而不是 k 的增大。

## 5. 示例计算 (忽略手续费简化)

### 假设池子

x = 1000 ETH, y = 400,000 USDT, k = 1000 * 400,000 = 400,000,000

### 瞬时价格

P = y / x = 400,000 USDT / 1000 ETH = 400 USDT/ETH

### 用户想用 Δx = 1 ETH 购买 USDT (φ = 0 简化)

Δx' = 1 ETH

Δy = (y *Δx') / (x + Δx') = (400,000* 1) / (1000 + 1) ≈ 399.6004 USDT

有效价格 = Δy / Δx = 399.6004 USDT / 1 ETH ≈ 399.6004 USDT/ETH

滑点 = (400 - 399.6004) / 400 ≈ 0.000999 ≈ 0.1%

### 如果用户想用 Δx = 100 ETH 购买 USDT

Δx' = 100 ETH

Δy = (400,000 * 100) / (1000 + 100) = 40,000,000 / 1,100 ≈ 36,363.636 USDT

有效价格 = 36,363.636 USDT / 100 ETH = 363.636 USDT/ETH

滑点 = (400 - 363.636) / 400 ≈ 0.091 ≈ 9.1%

## 6. Uniswap V3 的集中流动性 (影响 x 和 y)

在 V3 中，流动性提供者 (LP) 可以将资金集中在特定的价格区间内。这改变了 Swap 数学的“输入参数”：

### 有效储备金 (x_eff, y_eff)

当交易价格在某个 LP 头寸的价格区间 [P_low, P_high] 内时，该头寸的全部流动性才参与做市。否则，流动性会逐渐变成单一的 Token A 或 Token B。

### 分段计算

一次大额交易可能会穿越多个不同的 LP 价格区间。在每个价格区间内

- 使用该区间内当前活跃的流动性 (x_active, y_active) 计算。

- 仍然遵循 x * y = k （在特定价格区间内）。

- 当价格移动到一个区间的边界时，该区间的流动性可能耗尽或失效，下一个区间（如果有）的流动性开始生效。

### 更高的资本效率

在目标价格区间内，x_eff 和 y_eff 相对于 V2 的全局池更大，意味着在目标价格附近交易可以承受更大的交易量而滑点更低。一旦价格大幅偏离目标区间，有效流动性急剧下降，滑点会显著增加。

## 总结

Uniswap Swap 的数学核心是恒定乘积公式 x *y = k。用户支付的输入 Token 数量 (Δx) 扣除手续费 (Δx') 后加入池子，导致输出 Token 的储备金减少 (Δy)，减少量由公式 Δy = (y* Δx') / (x + Δx') 精确计算。交易会导致价格沿曲线移动，产生滑点，滑点大小主要取决于交易量占池子储备金的比例。Uniswap V3 通过集中流动性允许 LP 在特定价格区间提供更高的有效储备金 (x_eff, y_eff)，从而在该区间内降低滑点，但其 Swap 计算在单一价格区间内仍然遵循相同的恒定乘积原理。理解 Δy = (y * Δx') / (x + Δx') 及其推导过程是掌握 Uniswap Swap 数学的关键。
